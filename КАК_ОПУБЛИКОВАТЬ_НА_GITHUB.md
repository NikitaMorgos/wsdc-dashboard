# Как опубликовать проект на GitHub и выложить дашборд на сайт

Ниже — пошагово: создание репозитория, загрузка кода и включение сайта с дашбордами через GitHub Pages.

---

## 1. Аккаунт и репозиторий на GitHub

1. Зайдите на **[github.com](https://github.com)** и войдите (или зарегистрируйтесь).
2. Нажмите **«+»** в правом верхнем углу → **«New repository»**.
3. Заполните:
   - **Repository name** — например: `wsdc-dashboard` или `dashboard-content`.
   - **Description** — по желанию (например: «WSDC Points дашборды»).
   - **Public**.
   - Галочку **«Add a README file»** можно не ставить (README у вас уже есть в проекте).
4. Нажмите **«Create repository»**.

После этого откроется страница нового пустого репозитория. Не закрывайте её — понадобится ссылка на репозиторий (например: `https://github.com/ВАШ_ЛОГИН/wsdc-dashboard`).

---

## 2. Git в папке проекта (если ещё не настроен)

**Что такое Git здесь:**  
Git — это программа, которая «запоминает» состояние папки (какие файлы есть, что в них написано). Чтобы выложить проект на GitHub, папка должна быть **репозиторием Git** — то есть в ней один раз нужно выполнить команду `git init`. После этого Git начнёт отслеживать файлы в этой папке, и вы сможете отправить их на GitHub.

**Что нужно сделать по шагам:**

### 2.1. Открыть терминал именно в папке проекта

Терминал должен работать **в той папке**, где лежат файлы проекта: `upload_to_google_sheets.py`, `novice_closing_speed_dashboard.html`, `index.html`, папка `charts` и т.д.

**Вариант А — в Cursor / VS Code:**  
1. Откройте папку проекта в Cursor (File → Open Folder → выберите папку `Dashboard_content`).  
2. Меню **Terminal** → **New Terminal** (или сочетание клавиш для терминала).  
3. Внизу откроется терминал, и он уже будет в папке проекта. Должна быть строка вида:  
   `C:\Users\user\Dropbox\Public\Cursor\Dashboard_content>`  
   или  
   `PS C:\Users\user\...\Dashboard_content>`

**Вариант Б — в проводнике Windows:**  
1. Откройте проводник и перейдите в папку проекта (например, `C:\Users\user\Dropbox\Public\Cursor\Dashboard_content`).  
2. В адресной строке вверху окна (где путь к папке) нажмите один раз и введите: `cmd` или `powershell`, затем Enter.  
3. Откроется окно терминала уже в этой папке.

**Вариант В — вручную:**  
1. Откройте **cmd** или **PowerShell** (поиск Windows: «cmd» или «PowerShell»).  
2. Перейдите в папку проекта командой (подставьте свой путь):  
   `cd C:\Users\user\Dropbox\Public\Cursor\Dashboard_content`

Проверка: в терминале выполните `dir` (или `ls` в PowerShell) — в списке должны быть файлы `index.html`, `upload_to_google_sheets.py` и т.д.

---

### 2.2. Проверить, установлен ли Git

В том же терминале введите:

```bash
git --version
```

- Если появится строка вида `git version 2.xx.x` — Git установлен, переходите к пункту 2.3.  
- Если пишет, что команда не найдена («'git' is not recognized» и т.п.) — Git нужно установить. Два варианта:

  **Вариант 1 — через программу с интерфейсом (проще):**  
  Скачайте **GitHub Desktop**: [desktop.github.com](https://desktop.github.com) — один установщик для Windows. Бесплатно, от GitHub. После установки откройте программу → **File** → **Add local repository** → укажите папку проекта.  
  **Если программа пишет «This directory does not appear to be a Git repository»** — нажмите ссылку **«create a repository here instead»** в этом же окне. GitHub Desktop создаст репозиторий в этой папке. После этого нажмите **«Add repository»** — папка откроется в программе. Дальше делайте шаг 3 через GitHub Desktop (см. раздел ниже «Шаг 3 в GitHub Desktop»).

  **Вариант 2 — через командную строку:**  
  1. Скачайте установщик с [git-scm.com](https://git-scm.com/download/win).  
  2. Установите (можно оставить настройки по умолчанию).  
  3. Закройте и снова откройте терминал, снова выполните `git --version`.

---

### 2.3. Понять, есть ли уже репозиторий Git в папке

В папке проекта в терминале выполните:

```bash
git status
```

**Если команда сработала** (появился текст вроде «On branch main» или «No commits yet» или список файлов) — репозиторий уже есть, **переходите к шагу 3** (первый коммит и привязка к GitHub).

**Если появилось сообщение вроде:**  
`fatal: not a git repository (or any of the parent directories)`  
— значит, в этой папке Git ещё не инициализирован. Тогда выполните один раз:

```bash
git init
```

После этого появится строка вида: `Initialized empty Git repository in .../Dashboard_content/.git`.  
Теперь эта папка — репозиторий Git. Переходите к шагу 3.

---

### 2.4. Зачем нужен файл `.gitignore`

В проекте уже есть скрытый файл **`.gitignore`**. В нём перечислены имена файлов и папок, которые **не должны** попадать на GitHub, например:

- `credentials.json`, `credentials.json.json` — ключи доступа к Google (их нельзя выкладывать в открытый доступ);
- служебные папки Python (`__pycache__`, `venv` и т.д.).

Git сам читает `.gitignore` и не будет добавлять в коммит эти файлы. Менять что-то в этом пункте не обязательно — просто знайте, что секреты в репозиторий не попадут.

Если не хотите заливать на GitHub большие CSV-файлы — откройте `.gitignore` в редакторе и добавьте в конец новую строку: `*.csv`. Тогда все файлы с расширением `.csv` тоже будут игнорироваться.

## 3. Первый коммит и привязка к GitHub

**Если при `git add .` или коммите появляется ошибка «Filename too long»** (слишком длинное имя файла, например у CSV):

1. Включите поддержку длинных путей в Git (в терминале в папке проекта, по одной строке):

   ```powershell
   git config core.longpaths true
   ```

2. Сбросьте индекс и добавьте файлы заново (файл `.gitignore` уже исключает `*.csv`, длинные CSV не попадут в коммит):

   ```powershell
   git reset
   git add .
   ```

3. Если снова ошибка — делайте коммит через **GitHub Desktop**: откройте репозиторий в программе, в списке изменений **снимите галочки** с всех файлов `.csv` (или оставьте только нужные файлы с галочками), введите сообщение коммита и нажмите **Commit to main**. CSV в коммит не попадут.

В той же папке проекта выполните по очереди:

```bash
git add .
git status
```

Посмотрите список файлов: не должно быть `credentials.json`, `credentials.json.json` и других секретов. Если что-то лишнее попало — добавьте это в `.gitignore` и снова `git add .`.

```bash
git commit -m "WSDC dashboards and scripts"
```

Подключите удалённый репозиторий (подставьте свой логин и имя репозитория):

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/wsdc-dashboard.git
```

Отправьте код на GitHub:

```bash
git branch -M main
git push -u origin main
```

Если GitHub попросит авторизацию — войдите (логин/пароль или токен, в зависимости от настроек). После успешного `git push` проект будет на GitHub.

---

### Шаг 3 в GitHub Desktop (если пользуетесь программой, а не терминалом)

1. После того как папка открыта в GitHub Desktop (вы нажали «create a repository here instead» и «Add repository»), слева появятся **Changes** — список изменённых файлов. Внизу слева введите сообщение коммита, например: **WSDC dashboards and scripts**, и нажмите синюю кнопку **«Commit to main»**.
2. Подключить этот репозиторий к вашему репозиторию на GitHub: меню **Repository** → **Repository settings** (или **Ctrl+,**). В поле **Primary remote repository (origin)** введите URL, например: `https://github.com/ВАШ_ЛОГИН/wsdc-dashboard.git` (подставьте свой логин и имя репозитория). Нажмите **Save**.
3. Вверху нажмите **«Publish repository»** или **«Push origin»** — код отправится на GitHub. Если программа попросит войти в GitHub — авторизуйтесь. После успешной отправки проект будет на GitHub, переходите к шагу 4 (включить сайт).

## 4. Включить сайт (GitHub Pages)

1. В репозитории на GitHub откройте вкладку **«Settings»** (Настройки).
2. В левом меню найдите раздел **«Pages»** (в блоке «Code and automation» или «Build and deployment»).
3. В блоке **«Build and deployment»**:
   - **Source** выберите **«Deploy from a branch»**.
   - **Branch** выберите **main**, папку — **/ (root)**.
   - Нажмите **Save**.

Через 1–2 минуты сайт будет доступен по адресу:

**`https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПОЗИТОРИЯ/`**

Например: `https://johndoe.github.io/wsdc-dashboard/`

На этом URL будет открываться файл **`index.html`** (главная страница со ссылками на дашборды). Отдельные дашборды будут по адресам вида:

- `https://ВАШ_ЛОГИН.github.io/wsdc-dashboard/novice_closing_speed_dashboard.html`
- `https://ВАШ_ЛОГИН.github.io/wsdc-dashboard/novice_rating_dashboard.html`
- и т.д.

---

## 5. Обновление сайта после изменений

Когда вы обновили дашборды (перегенерировали HTML) или изменили код:

```bash
git add .
git commit -m "Update dashboards"
git push origin main
```

После `git push` GitHub Pages сам подхватит изменения; через 1–2 минуты сайт обновится.

## Краткий чеклист

- [ ] Создан репозиторий на GitHub (Public).
- [ ] В папке проекта выполнен `git init` (если репозитория ещё не было).
- [ ] В `.gitignore` нет исключений для `credentials.json` и других секретов (они не должны попадать в коммиты).
- [ ] Выполнены `git add .`, `git commit`, `git remote add origin ...`, `git push -u origin main`.
- [ ] В настройках репозитория включён **Pages**: Source = **Deploy from a branch**, ветка **main**, папка **/ (root)**.
- [ ] В корне проекта есть **index.html** (главная страница со ссылками на дашборды).

## Если репозиторий уже есть

Если проект уже залит в другой репозиторий:

1. Откройте этот репозиторий на GitHub.
2. **Settings** → **Pages** → **Source**: **Deploy from a branch**, ветка **main**, папка **/ (root)** → **Save**.
3. Убедитесь, что в корне есть **index.html**. Если его нет — добавьте (см. ниже) и сделайте `git add index.html`, `git commit`, `git push`.

Главная страница **index.html** со ссылками на дашборды уже добавлена в проект — после первого `git push` она будет открываться по основному URL сайта.
