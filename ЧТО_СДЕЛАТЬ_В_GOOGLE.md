# Что сделать в Google, чтобы скрипт мог заливать данные в таблицу

Чтобы скрипт мог сам выкладывать данные из CSV в Google Таблицу, нужно один раз настроить доступ. Ниже — только действия в Google (без кода).

---

## 1. Создать проект в Google Cloud

1. Откройте **[Google Cloud Console](https://console.cloud.google.com/)** и войдите в свой Google-аккаунт.
2. Вверху нажмите на выпадающий список проектов → **«Создать проект»** (или «New Project»).
3. Название проекта — любое, например: **WSDC Points Uploader**.
4. Нажмите **«Создать»**.

---

## 2. Включить API для таблиц и диска

1. В меню слева: **«API и сервисы»** → **«Библиотека»** (APIs & Services → Library).
2. В поиске введите **Google Sheets API** → откройте → нажмите **«Включить»**.
3. Вернитесь в «Библиотека», найдите **Google Drive API** → откройте → нажмите **«Включить»**.

Без этих двух API скрипт не сможет открыть таблицу и записать в неё данные.

---

## 3. Создать сервисный аккаунт (Service Account)

1. В меню: **«API и сервисы»** → **«Учётные данные»** (Credentials).
2. Нажмите **«+ Создать учётные данные»** → **«Сервисный аккаунт»**.
3. Заполните:
   - **Название**: например, `wsdc-uploader`
   - **Идентификатор** подставится сам.
4. Нажмите **«Создать и продолжить»**.
5. На шаге «Предоставить доступ» можно нажать **«Готово»** (роли для таблицы зададим отдельно).
6. Нажмите **«Готово»**.

Сервисный аккаунт — это «робот» с собственным email. Именно ему мы потом дадим доступ к вашей таблице.

---

## 4. Скачать ключ (credentials)

1. **Где найти список сервисных аккаунтов:**
   - В левом меню Google Cloud Console нажмите **«≡»** (три полоски), если меню свёрнуто.
   - Выберите **«IAM & Admin»** (или **«IAM и администрирование»**) → **«Service accounts»** (**«Сервисные аккаунты»**).
   - Откроется страница со списком сервисных аккаунтов проекта (таблица: имя, описание, email).
   
   *Альтернатива:* **«APIs & Services»** → **«Credentials»** (**«Учётные данные»**) — прокрутите страницу вниз до блока **«Service accounts»**; там будет таблица, можно нажать на нужную строку или «Manage service accounts» и перейти к тому же списку.
2. В списке сервисных аккаунтов **кликните по имени** созданного аккаунта (например, `wsdc-uploader`) — откроется его страница.
3. Перейдите на вкладку **«Ключи»** (Keys).
4. **«Добавить ключ»** → **«Создать новый ключ»**.
5. Тип ключа выберите **JSON**.
6. Нажмите **«Создать»** — файл JSON скачается на компьютер.

---

## 5. Положить ключ в папку проекта и переименовать

1. Найдите скачанный файл (обычно что-то вроде `project-name-xxxxx.json`).
2. **Переименуйте** его в **`credentials.json`**.
3. **Переместите** его в папку с проектом (туда же, где лежат `upload_to_google_sheets.py` и `upload_latest_to_sheets.py`).

Важно: файл `credentials.json` не выкладывайте в интернет и не добавляйте в публичный репозиторий — в нём ключ доступа.

---

## 6. Узнать email сервисного аккаунта

1. Откройте файл **`credentials.json`** в любом текстовом редакторе.
2. Найдите строку **`"client_email"`**. Рядом будет email вида:  
   `something@project-name.iam.gserviceaccount.com`  
3. Скопируйте этот email — он понадобится на следующем шаге.

---

## 7. Дать доступ к вашей Google Таблице

1. Откройте **вашу Google Таблицу** (ту, куда должны заливаться данные).
2. Нажмите кнопку **«Настройки доступа»** / **«Поделиться»** (Share) в правом верхнем углу.
3. В поле **«Добавить пользователей»** вставьте скопированный **email из `client_email`**.
4. Права выберите **«Редактор»** (Editor), чтобы скрипт мог записывать и обновлять данные.
5. Снимите галочку «Уведомить пользователей», если не хотите отправлять письмо (сервисный аккаунт всё равно не читает почту).
6. Нажмите **«Готово»** или **«Отправить»**.

После этого скрипт сможет открыть эту таблицу по ID и записать в неё данные.

---

## 8. Узнать ID таблицы (если будет своя таблица)

1. Откройте нужную Google Таблицу в браузере.
2. Посмотрите адрес в строке URL. Он выглядит так:  
   `https://docs.google.com/spreadsheets/d/ **ЭТО_ID_ТАБЛИЦЫ** /edit...`
3. Скопируйте длинную строку между `/d/` и `/edit` — это **ID таблицы**.

В скриптах по умолчанию уже прописан ID:  
`1ZaJXXCGQl8a1nrn1Bq6Wf2HjMI3dUGVLdb5vHT0P3LE`  
Если используете другую таблицу — передайте её ID при запуске (см. ниже).

---

## 9. Установить библиотеки (один раз на компьютере)

В терминале (в папке проекта) выполните:

```bash
pip install gspread google-auth
```

После этого скрипты загрузки будут готовы к работе.

---

## Как потом заливать данные

**Вариант 1 — залить последний файл (со всеми комментариями и полями):**

```bash
python upload_latest_to_sheets.py
```

Скрипт сам найдёт файл `*with_events_months_fixed.csv`, очистит лист «WSDC Points» и загрузит в него данные.

**Вариант 2 — залить конкретный CSV:**

```bash
python upload_to_google_sheets.py "путь_к_файлу.csv" --clear
```

**Вариант 3 — указать свою таблицу и лист:**

```bash
python upload_latest_to_sheets.py "файл.csv" ВАШ_SHEET_ID "Имя листа"
```

или:

```bash
python upload_to_google_sheets.py "файл.csv" ВАШ_SHEET_ID "Имя листа" A1 --clear
```

- [ ] Запущен `python upload_latest_to_sheets.py` (или с нужными аргументами)

Если что-то не сработает — проверьте, что не пропущен шаг 2 (включение API) или шаг 7 (доступ к таблице для email из `credentials.json`).
