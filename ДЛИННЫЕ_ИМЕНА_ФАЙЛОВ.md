# Как победить длинные названия файлов (CSV и Git)

Длинные имена вроде `wsdc_points_2026-01-28_updated_with_pdf_..._fixed_22030_with_events_months_fixed.csv` вызывают ошибку **«Filename too long»** в Windows и Git. Ниже — что сделать сейчас и как избежать проблемы в будущем.

---

## 1. Что сделать сразу (чтобы коммит прошёл)

### Вариант А: Терминал

В папке проекта выполните **по одной команде**:

```powershell
git config core.longpaths true
```

```powershell
git reset
```

```powershell
git add .
```

Дальше делайте коммит как обычно. В `.gitignore` уже есть `*.csv` — CSV в коммит не попадут.

### Вариант Б: GitHub Desktop

1. Откройте репозиторий в GitHub Desktop.
2. В списке **Changes** снимите галочки со **всех** файлов с расширением `.csv`.
3. Оставьте галочки на остальных файлах (`.gitignore`, HTML, скрипты и т.д.).
4. Введите сообщение коммита и нажмите **Commit to main** → **Push origin**.

Так длинные CSV не попадут в репозиторий, и ошибка не появится.

---

## 2. На будущее: короткое имя для «итогового» файла

Чтобы не упираться в лимит длины имени и не зависеть от длинных имён в скриптах:

1. **Скопируйте итоговый CSV под коротким именем** (один раз после пайплайна или по желанию):

   - Файл с длинным именем:  
     `wsdc_points_2026-01-28_..._with_events_months_fixed.csv`
   - Скопируйте его и назовите копию, например: **`wsdc_latest.csv`**

2. **Скрипты уже умеют брать короткое имя:**  
   `upload_latest_to_sheets.py` и `append_points_column_to_sheets.py` сначала ищут **`wsdc_latest.csv`** в папке проекта. Если он есть — используют его; если нет — ищут файл по маске `*with_events_months_fixed.csv`.

То есть вы можете:
- хранить длинное имя для истории (оно по‑прежнему в `.gitignore` и не лезет в Git);
- для загрузки в таблицы и дашбордов всегда вызывать скрипты без аргументов — они возьмут `wsdc_latest.csv` или последний подходящий CSV.

**Как быстро создать короткую копию вручную:**  
Проводник → нужный CSV → Ctrl+C → Ctrl+V → переименовать копию в `wsdc_latest.csv`.

Или одной командой в PowerShell (в папке проекта, подставьте своё длинное имя если нужно):

```powershell
Copy-Item ".\*with_events_months_fixed.csv" -Destination ".\wsdc_latest.csv"
```

(Скопируется первый найденный файл с таким окончанием.)

---

## 3. Кратко

| Проблема | Решение |
|----------|--------|
| Git выдаёт «Filename too long» при коммите | Включить длинные пути: `git config core.longpaths true`, не добавлять CSV в коммит (`.gitignore` + снять галочки с CSV в GitHub Desktop или не добавлять их в индекс). |
| Хочу не зависеть от длинных имён в скриптах | Держать копию итогового файла под именем `wsdc_latest.csv`; скрипты загрузки и добавления столбца уже поддерживают это имя. |
