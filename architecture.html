Архитектура проекта WSDC Points

Схема данных, компонентов и публикации

Данные об участниках и их WSDC ID берутся напрямую из API DanceConvention по итогам ивентов. Затем по ID запрашиваются очки в WSDC Registry. Итоговый CSV сразу идёт в дашборды — Google Таблица в процессе больше не участвует.

1. Схема «от данных до сайта»

ИСТОЧНИКИ ДАННЫХ

WSDC Registry API (points.worldsdc.com) — очки по WSDC ID
PDF/TXT — ручные списки с ивентов (резервный путь)

add_dancers_from_pdf.py — имена из PDF/TXT → поиск в WSDC API
ОЧИСТКА И ОБОГАЩЕНИЕ
fix_*, add_novice_*, add_event_info.py, finalize_data.py
ИТОГОВЫЙ CSV
name_ru, name_en, wsdc_id, division, role, points_YYYY-MM-DD, last_event_*, months_to_close_novice …
ВИЗУАЛИЗАЦИЯ
create_all_divisions_dashboard.py
create_novice_*, create_advanced_charts.py → HTML (Chart.js)
ПУБЛИКАЦИЯ САЙТА

3. Основные компоненты по ролям

Сбор данных

  • export_dc_wsdc_from_events.py — обращается к DanceConvention API, скачивает списки участников финалов (и prelim), извлекает WSDC ID → CSV
  • fetch_wsdc_registry_from_dc_export.py — принимает CSV с WSDC ID, запрашивает актуальные очки в WSDC Registry API, обогащает данные (дивизион, роль, ивенты)
  • add_dancers_from_pdf.py — резервный путь: парсит имена из PDF/TXT, ищет по WSDC API
  • add_event_info.py — даты/названия ивентов и months_to_close_novice из API

Очистка и нормализация

  • fix_* — исправление имён, ID, дублей
  • add_novice_columns.py, add_novice_zero_points.py — колонки Novice/Allowed, строки с 0 очков
  • finalize_data.py — финальный вид CSV (points_<дата>, name_ru, порядок колонок)

Визуализация

  • create_all_divisions_dashboard.py — общий рейтинг по всем дивизионам
  • create_novice_rating_dashboard.py, create_novice_closing_speed_dashboard.py — рейтинг и скорость закрытия Novice
  • create_advanced_charts.py — дополнительные аналитические дашборды

Публикация

  • index.html — главная страница со ссылками на дашборды
  • Репозиторий на GitHub + GitHub Pages (ветка main) → публичный URL сайта

5. Типовой сценарий использования

  1. Получить участников ивента из DC — export_dc_wsdc_from_events.py → CSV с WSDC ID.
  2. Запросить очки из WSDC — fetch_wsdc_registry_from_dc_export.py --input <csv> → обогащённый CSV с дивизионами, ролями, очками.
  3. Почистить и финализировать — при необходимости запустить fix_*, add_novice_*, add_event_info.py, finalize_data.py.
  4. Пересобрать дашборды — create_all_divisions_dashboard.py и другие create_*, передав итоговый CSV.
  5. Опубликовать сайт — в GitHub Desktop: Commit to main → Push origin. Через 1–2 минуты GitHub Pages обновит сайт.

6. Зависимости и секреты

  • Python: pandas, requests; для дашбордов — Chart.js из CDN.
  • DanceConvention: учётные данные DC (DC_EMAIL, DC_PASSWORD) передаются через переменные окружения или .env.
  • Git ignore: *.csv, .env, кэш Python — в .gitignore.
