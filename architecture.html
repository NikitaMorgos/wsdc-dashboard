Архитектура проекта WSDC Points

Схема данных, компонентов и публикации

Сбор данных об очках WSDC из реестра и Google Таблиц, очистка и обогащение, визуализация в дашбордах и публикация на сайте; синхронизация с Google Таблицей.

3. Основные компоненты по ролям

Сбор и обогащение

  • wsdc_from_google_sheet.py — Google Sheet → транслит → WSDC API → CSV
  • add_dancers_from_pdf.py — добавление имён из PDF/TXT и запрос очков в WSDC
  • add_event_info.py — даты/названия ивентов и months_to_close_novice из API

Очистка и нормализация

  • fix_* — исправление имён, ID, дублей; apply_google_sheet_comments.py — правила из комментариев в таблице
  • add_novice_columns.py, add_novice_zero_points.py — колонки Novice/Allowed, строки с 0 очков
  • finalize_data.py — финальный вид CSV (points_<дата>, name_ru, порядок колонок)

Визуализация

  • create_novice_rating_dashboard.py, create_novice_closing_speed_dashboard.py — рейтинг и скорость закрытия Novice
  • create_novice_dashboard.py, create_novice_leader_charts.py, create_advanced_charts.py — дашборды в charts/*.html

Интеграция с Google

  • upload_to_google_sheets.py — загрузка CSV в лист (с опцией --clear)
  • upload_latest_to_sheets.py — загрузка последнего CSV или wsdc_latest.csv
  • append_points_column_to_sheets.py — добавление столбца points_<дата> в таблицу

Публикация

  • index.html — главная страница со ссылками на дашборды
  • Репозиторий на GitHub + GitHub Pages (ветка main) → публичный URL сайта

5. Типовой сценарий использования

  1. Обновить данные с WSDC — запустить wsdc_from_google_sheet.py или add_dancers_from_pdf.py, затем цепочку fix/apply/finalize при необходимости.
  2. Получить итоговый CSV — результат пайплайна (например *with_events_months_fixed.csv). Опционально скопировать в wsdc_latest.csv.
  3. Обновить Google Таблицу — upload_to_google_sheets.py <csv> --clear или upload_latest_to_sheets.py, либо только новый столбец: append_points_column_to_sheets.py.
  4. Пересобрать дашборды — запустить нужные create_*_dashboard.py / create_*_charts.py, передав итоговый CSV.
  5. Опубликовать изменения на сайте — в GitHub Desktop: рабочая папка → Commit to main → Push origin. Через 1–2 минуты GitHub Pages обновит сайт.

6. Зависимости и секреты

  • Python: pandas, requests, beautifulsoup4, gspread, google-auth; для дашбордов — pandas + Chart.js из CDN.
  • Секреты: credentials.json (или credentials.json.json) для Google Sheets API; не коммитить (см. ЧТО_СДЕЛАТЬ_В_GOOGLE.md).
  • Игнор в Git: *.csv, *credentials*.json, кэш Python — в .gitignore; короткое имя wsdc_latest.csv (см. ДЛИННЫЕ_ИМЕНА_ФАЙЛОВ.md).
